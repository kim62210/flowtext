# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue in Flowtext, please do not open a public issue first.

Instead, report it privately to the maintainer through the repository security advisory workflow or the primary maintainer contact channel listed on GitHub.

For this repository, the preferred first channel is the GitHub security advisory workflow. If that is not available, contact the maintainer privately through **@kim62210** on GitHub.

Please include:

- a short description of the issue
- impact assessment
- reproduction steps or a proof of concept
- affected versions or commit range if known

## Response Expectations

Flowtext is currently a small, early-stage project, so response times may vary. Valid reports will be acknowledged and triaged as quickly as practical.

Security reports and code-of-conduct reports are handled through different channels. Conduct concerns should follow `CODE_OF_CONDUCT.md`, not this document.

## Scope Notes

At this stage, the highest-priority reports are issues involving:

- unsafe parsing or execution behavior
- supply-chain or dependency-risk concerns
- incorrect assumptions that could lead consumers to trust layout guarantees that are not actually provided
